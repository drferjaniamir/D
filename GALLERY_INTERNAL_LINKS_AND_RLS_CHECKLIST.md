# Checklist: Gallery Internal Links & RLS Security Fixes

This checklist outlines the steps to make internal links functional on the `/transformation` page and to secure the database tables by enabling Row Level Security (RLS).

---

## 1. Database Security (RLS Setup)
Execute the following SQL script in the **Supabase Dashboard SQL Editor** to secure the tables and allow public read access (SELECT):

```sql
-- 1. Enable Row Level Security (RLS) on all exposed tables
ALTER TABLE public.services ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.sub_services ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.internal_links ENABLE ROW LEVEL SECURITY;

-- 2. Create public SELECT policies so the website can read them
CREATE POLICY "Allow public read access for services" 
ON public.services FOR SELECT 
USING (true);

CREATE POLICY "Allow public read access for sub_services" 
ON public.sub_services FOR SELECT 
USING (true);

CREATE POLICY "Allow public read access for internal_links" 
ON public.internal_links FOR SELECT 
USING (true);
```

- [x] **Step 1.1**: Run the SQL script above in your Supabase SQL Editor.
- [x] **Step 1.2**: Verify that the tables are secure and the website still fetches services successfully.
